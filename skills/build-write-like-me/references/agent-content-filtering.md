# Filtering out content the person didn't write

A voice profile is poisoned by text the person didn't actually write. In the
agent era, a lot of "their" PRs and comments are partly or wholly AI-generated.
Exclude that content before analyzing, and definitely before quoting it as their
voice. These are heuristics, not laws; tag aggressively, then eyeball borderline
cases.

## What to exclude

1. **Comments/replies made on their behalf by a coding agent.**
2. **PR descriptions for agent-written code** (the body was likely AI-drafted too).
3. **AI-edited passages** inside otherwise-human docs.

## Reliable tells of agent-authored content

**Explicit attribution markers** (highest confidence):
- "🤖", "Generated with Claude Code", "Co-Authored-By: Claude", "Co-authored-by:
  Copilot", "Generated by Copilot", "claude.com/claude-code", "noreply@anthropic",
  bot account logins (e.g. `*-bot`, `svc-*`, MAE/review-bot accounts),
- "### Code review" auto-review blocks.

**The auto-reply signature** (very common, no explicit marker):
> "Good catch — fixed in `a1b2c3d`. The function now does X, mirroring the
> pattern in Y, and I added a test covering Z."

The cadence is: acknowledge (Good catch / Good point / Addressed / Fixed in) +
a commit SHA + a multi-sentence mechanism explanation, often with an em-dash.
A human's real reply to the same comment is usually "done" or "updated". Flag a
comment as agent-authored if it matches `^(good catch|good point|great point|fair
point|addressed|fixed in|done in)` AND (contains a 7-40 char hex SHA OR is long),
or if it contains an em-dash together with a commit SHA, or is a long mechanism
dump citing a SHA.

**Templated PR bodies:**
- A generated "# Problem & Solution Overview" / "## Summary" wall of three
  polished, parallel-structured paragraphs, especially with em-dashes. Hand-
  written descriptions tend to be terser and plainer.

**Em-dashes and semicolons as an AI tell:**
- Most people's raw typing has very few em-dashes. If a person's short messages
  and code-review comments have ~zero em-dashes but a PR body or doc passage is
  sprinkled with them, that passage was likely AI-drafted or AI-edited. Use the
  person's own *short, clearly-typed* messages to establish their baseline, then
  treat em-dash-heavy long-form as suspect.

## Pasted vs. typed

People paste things into chats and docs: blog drafts, other people's text, AI
output they're reviewing. A pasted block is not their voice even though it sits in
"their" message. Heuristics:
- Quoted/fenced blocks (```), block quotes, and text introduced with "here's the
  draft:" / "made this change:" are usually pasted.
- Compute stylometrics separately for short own-typed messages vs. long pasted/
  quoted material; if they diverge sharply (e.g. em-dashes only in the long
  stuff), trust the short own-typed baseline for mechanics.

## How to use this

The bundled scripts tag suspected agent content with a flag rather than deleting
it, so you can inspect what was filtered and adjust the threshold. Before quoting
any sentence as the person's voice, make sure it passed the filter. When in
doubt, prefer their **short, casual, clearly-typed** material — it is the least
likely to be contaminated.
